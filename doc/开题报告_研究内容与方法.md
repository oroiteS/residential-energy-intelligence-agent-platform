# 二、研究（或设计）的主要内容和方法

## 1. 研究目标和研究内容

### 1.1 研究目标

本研究旨在设计并实现一个面向居民场景的用电分析与节能建议智能系统。系统以居民用电时序数据为核心输入，围绕“数据预处理、统计分析、行为识别、负荷预测、结果可视化、节能建议生成”构建完整的端到端技术链路。在方法上，项目采用规则分析与深度学习结合的思路：一方面通过统计分析和规则标签构建可解释的基础分析能力，另一方面引入时序深度学习模型提升用电行为识别与负荷预测的智能化水平；同时结合大语言模型问答能力，将结构化分析结果转化为面向用户的自然语言解释与节能建议。

本课题的目标不仅在于完成一个可运行的居民用电分析系统，还在于体现时序数据建模、模型训练与推理部署、前后端协同集成以及智能代理应用等方面的综合实践能力。最终系统应能够帮助用户理解自身用电规律，识别典型用电行为特征，预测短期负荷变化趋势，并输出可执行、可解释的节能建议，从而兼顾研究价值、工程完整性与实际应用意义。

### 1.2 研究内容

（1）系统需求分析与总体方案设计。围绕居民用电分析与节能建议场景，梳理系统的功能边界、核心流程与非功能需求，明确数据导入、数据管理、统计分析、行为分类、负荷预测、节能建议、智能问答、结果可视化与报告导出等模块的职责划分。结合项目实际工程约束，形成“Go 主服务 + Python 模型与智能体服务 + React 前端”的分层架构方案，并完成接口边界、数据流转关系和部署方式设计。

（2）数据预处理与基础分析模块研究与实现。针对居民用电原始数据，设计统一的数据导入与清洗流程，支持 CSV/Excel 格式文件读取、时间字段解析、异常记录剔除、缺失值处理与 `15min` 粒度时间对齐。围绕户级总负荷与电器子表信息，提取基础统计指标，并进一步构造 `active_appliance_count` 与 `burst_event_count` 等辅助特征。在此基础上实现总用电量、日均用电量、峰谷平占比、趋势分析、典型日曲线等功能，并通过可视化图表展示分析结果。

（3）居民用电行为分类与负荷预测模型研究。针对居民用电时序具有周期性、波动性与个体差异明显等特点，建立面向单日负荷曲线的行为分类任务和面向短期未来负荷的预测任务。分类部分以规则方法生成 4 类稳定标签，并采用 TCN 模型完成日级行为模式识别；预测部分基于最近 3 天的历史序列，分别构建 Seq2Seq LSTM 与 Transformer 两类模型，对下一天 96 个时间步的总负荷进行预测。研究内容包括数据集构造、模型结构设计、训练策略、评估指标与实验对比分析。

（4）节能建议生成与智能问答能力集成。面向用户理解与交互需求，构建由规则建议与大语言模型问答组成的双层建议机制。系统首先基于统计分析结果、行为分类结果和预测摘要生成可解释的规则节能建议；在此基础上，通过 LangChain 组织单体问答智能模块，将结构化上下文与用户问题结合，生成更具针对性的自然语言建议与解释。该部分重点研究提示词设计、上下文约束、回答可解释性以及服务降级策略，以提高建议质量与系统稳定性。

（5）全栈系统开发与模块联调。围绕实际可用的毕业设计系统目标，完成前后端分离架构下的功能集成。Go 主服务负责数据集管理、统计分析、报告导出以及对 Python 服务的统一调用；Python Robyn 服务负责分类、预测与智能问答接口；前端界面负责上传数据、展示分析结果、呈现图表和支持问答交互。通过 REST API 完成模块协同，打通从数据上传到结果输出的完整链路。

（6）系统测试、评估与优化。面向研究成果的可靠性与可演示性，制定覆盖功能正确性、接口联调、模型效果、推理性能与用户体验的测试方案。分类模型重点评估准确率与 F1 值，预测模型重点评估 MAE、RMSE、SMAPE、WAPE 等指标，并结合系统响应时间、推理效率和异常场景降级表现进行综合分析。根据测试结果持续优化系统实现、模型参数和交互细节，形成完整的实验结论与文档材料。

## 2. 拟采取的研究方法、技术路线和实验方案

### 2.1 研究方法

本课题采用“需求驱动、分层实现、迭代优化、模型与系统协同验证”的研究方法。首先，从居民用电分析的真实应用场景出发，明确系统必须解决的数据处理、行为识别、结果解释与建议生成等关键问题，以需求分析指导后续模块设计。其次，在实现路径上采用分层递进方式，优先完成数据预处理、统计分析和可视化等基础能力，保证数据链路闭环与结果口径一致；随后再引入深度学习模型开展行为分类与负荷预测研究，在稳定的数据基础上完成模型训练、评估与推理集成；最后加入智能问答模块，实现结构化分析结果向自然语言交互的转化。

在模型研究方法上，分类任务采用“规则标注 + 监督学习”的方式构建训练体系，以规则生成稳定、可解释、可复现的居民用电行为标签，再通过 TCN 模型学习日级负荷模式；预测任务采用“多模型对比”的方式，以 Seq2Seq LSTM 和 Transformer 为主要研究对象，对短期负荷预测能力进行实验比较。整个过程坚持实验可复现原则，固定数据处理口径、样本构造方式、模型输入输出格式和核心评估指标，确保研究结果具备可验证性。

在系统实现方法上，课题采用敏捷迭代思想组织开发过程，以可运行、可演示、可测试为每阶段目标。每个阶段均同步推进模块实现、接口联调和结果验证，避免只完成单点算法而忽视整体系统落地。对于智能问答部分，采用“结构化上下文约束 + 失败降级”的方法控制回答质量与工程风险，使大语言模型能力服务于分析解释与建议生成，而不是脱离数据上下文进行泛化回答。

### 2.2 技术路线

本课题的技术路线按照“离线建模 + 在线推理 + 前后端集成”的思路展开，整体分为数据层、模型层、服务层和交互层四个部分。

首先，在数据层，系统对居民用电原始数据执行统一预处理。原始数据经过格式解析、时间对齐、缺失处理、异常记录剔除和特征构造后，被组织为 `15min` 粒度的基础时序数据，并进一步生成分类任务数据集与预测任务数据集。其中，分类任务以单天 `96 x 5` 时序样本作为输入，五个通道为 `Aggregate`、`slot_sin`、`slot_cos`、`weekday_sin`、`weekday_cos`；预测任务以最近 3 天共 `288` 个时间步的历史序列作为输入、以下一天 `96` 个时间步的总负荷作为预测目标。

其次，在模型层，基于 PyTorch 构建两类核心模型。行为分类部分采用 TCN 模型，对居民单日负荷曲线进行 4 类行为模式识别；负荷预测部分采用 Seq2Seq LSTM 与 Transformer 两种时序模型，对未来一天负荷进行预测，并通过统一评估指标比较模型效果。模型训练以离线脚本方式完成，训练完成后将权重接入推理服务，实现本地部署与在线调用。

再次，在服务层，系统采用双后端分工协作的架构。Go + Gin 主服务负责数据集管理、统计分析、业务编排、结果持久化与报告导出；Python + Robyn 服务负责承载分类推理、预测推理与基于 LangChain 的智能问答能力。两者之间通过 REST 风格的 HTTP/JSON 接口通信，不引入复杂消息队列或多智能体编排机制，从而在保证工程清晰度的前提下完成完整功能闭环。

最后，在交互层，前端采用 React + TypeScript 构建单页应用，结合 ECharts 展示负荷趋势、行为分类结果、预测曲线与节能建议。系统将统计分析结果、分类结果和预测摘要统一组织后，一方面生成规则化节能建议，另一方面作为上下文输入智能问答模块，支持用户围绕用电结构、预测趋势和节能策略进行交互式查询。通过这一技术路线，最终形成一个集数据处理、时序建模、可视化展示与智能建议于一体的居民用电分析系统。

从实验方案看，课题将围绕三个层面展开验证：一是验证数据预处理与统计分析链路的正确性，确保数据样本构造与指标计算稳定可靠；二是验证 TCN 分类模型与 Seq2Seq LSTM、Transformer 预测模型的效果差异，形成分类与预测任务的实验结果；三是验证系统集成后的接口可用性、推理响应速度、建议生成质量与异常场景下降级能力，最终支撑毕业设计系统演示与论文撰写。
