# 前端开发规划文档

> 项目：居民用电分析与节能建议系统
>  
> 适用范围：`frontend/` React 前端
>  
> 当前目标：先完成可联调的仪表盘主流程，再逐步补齐预测、问答、报告与系统配置能力

---

## 1. 文档目标

本文件用于固定前端开发的执行口径，避免后续开发过程中出现以下问题：

- 页面范围不断膨胀，首版迟迟无法形成闭环
- 页面字段命名与接口契约不一致，联调反复返工
- 图表、分类文案、降级态等展示规则没有统一标准
- 缺少任务进度记录，后续难以快速判断已完成内容

本文件的作用不是替代需求文档和 API 文档，而是回答三个问题：

1. 前端当前具体负责什么
2. 页面、组件、接口应按什么顺序落地
3. 如何通过勾选机制持续跟踪完成度

任务跟踪约定：

- `- [x]` 表示已完成并已落地
- `- [ ]` 表示尚未完成或仅停留在设计阶段
- 每完成一个页面、模块、接口联调点，应同步更新本文件中的勾选状态
- 若某项已完成但存在阻塞问题，应在该项下补充一句简短说明

---

## 2. 前端定位

React 前端是系统的交互与展示层，负责：

- 提供数据集上传、列表、详情查看入口
- 展示统计卡片、趋势图、峰谷图、分类结果、预测结果与建议内容
- 提供问答弹窗、报告导出入口与必要的配置界面
- 处理加载态、空状态、错误态、降级态
- 将稳定英文枚举映射为中文展示文案

前端不负责：

- 统计指标二次计算
- 分类或预测模型推理
- 规则建议生成逻辑本身
- LLM 提示词拼装与上下文组装

边界结论：

- 图表数据、统计摘要、分类结果、预测摘要由后端直接返回
- 前端主要负责展示编排、交互流程和状态反馈

## 3. 文档优先级与执行口径

当前前端实现应优先遵循以下文档：

1. `doc/chinese_project.md`
2. `doc/api_design.md`
3. `doc/后端.md`
4. `doc/model_data_pipeline.md`

若出现不一致，按以下规则处理：

- 功能范围以 `chinese_project.md` 为准
- 接口字段与响应结构以 `api_design.md` 为准
- 联调阶段顺序与当前执行口径以 `后端.md` 为准
- 分类标签、预测窗口等建模展示语义以 `model_data_pipeline.md` 为准

### 3.1 当前冻结口径

- 前端框架固定为 `React + TypeScript + Vite`
- 图表库固定为 `ECharts`
- 包管理固定为 `pnpm`
- 系统是单用户、本地部署、零登录
- 不做异常告警模块
- 不做多用户登录、权限系统
- 不做多种 LLM 原生协议适配

### 3.2 当前已冻结的展示规则

- 分类标签仅允许以下 4 个稳定值：
  - `day_high_night_low`
  - `day_low_night_high`
  - `all_day_high`
  - `all_day_low`
- 前端必须负责将上述英文标签映射为中文文案
- 智能问答失败时，必须展示规则建议与降级提示，不能阻塞主流程
- 图表数据优先直接消费后端返回结果，不把复杂聚合逻辑堆到前端

### 3.3 当前已修正的执行口径

`model_history_window_config` 当前统一按以下口径执行：

- `classification_days = 1`
- `forecast_history_days = 3`

说明：

- 不再使用过时的 `7 / 7` 示例值
- 相关文档与初始化脚本已同步修正，前端不要写死旧值

## 5. 推荐目录划分

建议前端采用清晰的分层结构，避免页面逻辑、接口调用和展示组件混杂。

推荐目录：

```text
frontend/
├── src/
│   ├── app/
│   ├── pages/
│   │   ├── datasets/
│   │   ├── dataset-detail/
│   │   └── settings/
│   ├── components/
│   │   ├── charts/
│   │   ├── cards/
│   │   ├── forms/
│   │   └── feedback/
│   ├── services/
│   ├── hooks/
│   ├── types/
│   ├── constants/
│   └── utils/
└── public/
```

职责建议：

- `pages`：页面级编排
- `components/charts`：ECharts 图表组件
- `services`：按接口域拆分请求函数
- `types`：接口 DTO、响应结构、枚举类型
- `constants`：分类标签中文映射、状态文案映射等

## 10. 当前结论

前端现在不应该一开始就扩展过多页面，而应先完成以下三件事：

1. 固定页面结构、类型定义与中文映射规则
2. 先把上传、列表、详情、分析图表这条主流程跑通
3. 再逐步接入分类、建议、问答、预测、报告

只要后续持续维护本文件中的勾选状态，就可以快速判断当前完成度、剩余工作和联调阻塞点。
